# はじめに
Copilot CLI を使っていて「premium requests 使いすぎてないかな...」ってなることありませんか？私は学生プランで GitHub Copilot を利用しているのですが、`Remaining requests: xx.x%` という表示を見るたびに「えーっと今日はx日だから...」という具合に premium requests の消費ペースを気にしてしまいます。

そこで cpm(copilot premium requests) というツールを作ってみました。
![cpm のデモ](/images/demo.png)

プログレスバー以外にもオプションからテキストやグラフを選べます。
![テキストで premium requests の消費量を表示](/images/cpm_text.png)
![グラフで premium requests の消費量を表示](/images/cpm_graph.png)

# 使い方
cargo でインストールします。
```sh
cargo install --git https://github.com/tknkaa/cpm
```
GitHub CLI で認証します。
```sh
gh auth login
```
デフォルトでは `gh auth token` からトークンを取得して、GitHub の API を叩くようにしています。オプションから premium requests の残りのパーセンテージを渡すこともでき、その場合はトークンを利用しません。
こんな感じで使えます。
```sh
# Fetch quota automatically via GitHub API
cpm

# Choose a display style
cpm --style progress  # default
cpm --style text
cpm --style graph

# Skip API fetch and specify remaining percentage directly
cpm --premium 23.4
```
