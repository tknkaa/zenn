# はじめに
Copilot CLI を使っていて「premium requests 使いすぎてないかな...」ってなることありませんか？私は学生プランで GitHub Copilot を利用しているのですが、`Remaining requests: xx.x%` という表示を見るたびに「えーっと今日はx日だから...」という具合に premium requests の消費ペースを気にしてしまいます。

そこで cpm(copilot premium monitor) というツールを作ってみました。

![cpm のデモ](/images/cpm.png)

プログレスバー以外にもオプションからテキストで表示することもできます。

![cpm のテキストのデモ](/images/cpm_text.png)

下の方に表示されるメッセージは `(消費した premium requests の割合) - (月の経過日数の割合)` が 15% を超えていたら `Overusing`、5% を超えていたら `Slightly fast`、-10% 未満だったら `Plenty left`、それ以外なら `Good pace` と表示しています。

# 注意事項
本ツールは GitHub Copilot の内部 API を使用しており、公式にサポートされていません。GitHub の仕様変更により予告なく動作しなくなる可能性があります。ご利用は自己責任でお願いします。

# 使い方
cargo でインストールします。
```sh
cargo install --git https://github.com/tknkaa/cpm
```

GitHub CLI で認証します。
```sh
gh auth login
```

デフォルトでは `gh auth token` からトークンを取得して、GitHub の内部 API を叩くようにしています。API の取得に失敗した場合は、パーセンテージを手動で入力するフォールバック画面が表示されます。`-p` オプションから直接パーセンテージを渡すこともできます。
```sh
# Fetch quota automatically via GitHub API
cpm

# Choose a display style
cpm --style progress  # default
cpm --style text

# Skip API fetch and specify remaining percentage directly
cpm -p 23.4
```
